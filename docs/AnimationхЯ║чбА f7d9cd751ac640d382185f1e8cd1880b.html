Animation基础

可以通过下面这个网站定制很多不错的动画效果。

理解CSS动画基础

为了更好的掌握CSS动画（animations），首先需要理解以下基本概念：

  • 关键帧（Keyframes）:使用@keyframes规则定义动画序列中的关键点，指定在动画过程中某个时刻应用的样式
  • 动画属性（Animations Properties）：包括animation-name、animation-duration、animation-timing-function、animation-delay、animation-iteration-count、animation-direction、animation-fill-mode和animation-play-state等属性，用于配置动画的不同方面。

学习关键帧的使用

创建动画的第一步是定义关键帧，例如：

@keyframes example {
from { background-color: red; }
to { background-color: yellow; }
}

或者使用百分比定义多个关键点：

@keyframes example {
  0% { background-color: red; }
  50% { background-color: blue; }
  100% { background-color: yellow; }
}

应用动画属性

定义了关键帧后，需要将动画应用到元素上，并配置动画属性：

div {
  animation-name: example; /* 引用@keyframes定义的动画名称 */
  animation-duration: 2s; /* 动画持续时间 */
  animation-timing-function: ease-in-out; /* 动画速度曲线 */
  animation-delay: 0s; /* 动画开始前的延迟 */
  animation-iteration-count: infinite; /* 动画循环次数 */
  animation-direction: alternate; /* 动画循环的方向 */
  animation-fill-mode: forwards; /* 动画状态的保持 */
  animation-play-state: running; /* 动画的播放状态 */
}

animation-timing-function

CSS animation-timing-function 属性值

animation-timing-function属性定义了动画的速度曲线，以下是一些常用的值：

  1. linear: 动画从头到尾的速度是相同的。
  1. ease: 默认值，动画以低速开始，然后加快，在结束前变慢。
  1. ease-in: 动画以低速开始。
  1. ease-out: 动画以低速结束。
  1. ease-in-out: 动画以低速开始和结束。
  1. step-start: 动画第一帧直接跳到结束状态，没有过渡效果。
  1. step-end: 动画一直保持开始状态，直到最后一帧突变到结束状态。

除了这些预定义的速度曲线之外，还可以使用cubic-bezier函数来自定义速度曲线：

  • cubic-bezier(n,n,n,n): 在贝塞尔曲线上自定义动画的变化速率，其中的四个值分别代表曲线的控制点。

还可以使用steps()函数指定步进间隔：

  • steps(n, start|end): 分割动画为n个步骤，通过第二个参数指定步骤开始的位置。

通过选择或自定义合适的animation-timing-function，可以实现更加丰富和自然的动画效果。

animation-iteration-count

animation-iteration-count属性定义了动画应该播放的次数。以下是可以设置的值：

  1. number: 可以定义动画应该播放多少次。例如，2表示动画将播放两次。
  1. infinite: 动画将无限次播放。

使用animation-iteration-count属性，可以控制动画的重复次数，无论是播放固定次数还是无限循环。

animation-direction

animation-direction属性定义了动画在每个周期内是否应该反向播放。以下是可以设置的值：

  1. normal: 动画每个周期内都正向播放。
  1. reverse: 动画每个周期内都反向播放。
  1. alternate: 动画交替正向和反向播放，第一次正向，第二次反向，以此类推。
  1. alternate-reverse: 动画交替反向和正向播放，第一次反向，第二次正向，以此类推。

这些值允许动画在播放时改变方向，从而创建更复杂和有趣的视觉效果。

animation-fill-mode

animation-fill-mode属性指定一个动画在开始之前和结束之后如何将样式应用于其目标。以下是可以设置的值：

  1. none: 不改变默认行为。动画不会应用任何样式到目标元素，在动画未执行或已完成时。
  1. forwards: 目标将保留由执行期间遇到的最后一个关键帧计算值。
  1. backwards: 在动画开始前，目标将应用第一个关键帧的样式，这种样式将在animation-delay期间持续。
  1. both: 动画将遵循forwards和backwards的规则，这意味着它将在两个方向上扩展动画的样式。

通过设置animation-fill-mode属性，可以控制动画在执行之前和之后如何影响元素的样式。

animation-play-state

animation-play-state属性控制动画的播放状态。以下是可以设置的值：

  1. running: 动画正在播放，这是默认值。
  1. paused: 动画被暂停。

使用场景举例

  1. 用户交互: 当用户与网页上的某个元素交互时，可能需要暂停动画。例如，当用户将鼠标悬停在一个滚动的图片轮播上时，可以通过设置animation-play-state: paused;来暂停轮播，以便用户能够更好地查看当前图片。
.carousel:hover {
  animation-play-state: paused;
}


2.
控制播放: 在一个网页上播放一段动画效果，当用户点击播放或暂停按钮时，可以通过JavaScript来切换animation-play-state的值，从而控制动画的播放和暂停。

// 假设有一个按钮用于控制动画
document.querySelector('#play-pause-button').addEventListener('click', function() {
  var animation = document.querySelector('#animated-element');
  if (animation.style.animationPlayState === 'paused') {
    animation.style.animationPlayState = 'running';
  } else {
    animation.style.animationPlayState = 'paused';
  }
});

在这两个场景中，通过更改animation-play-state属性，可以实现对动画播放的实时控制，提升用户的交互体验。
